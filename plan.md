# /plan

Use before any task touching more than 2 files, or any new feature.

1. Read chapter-1/SPEC.md and PROGRESS.md
2. Ask any clarifying questions before proceeding
3. Output a plan: what you will build, which files you will touch, in what order
4. Save the plan to .cursor/plans/[feature-name].md
5. Wait for approval before writing any code

Do not write a single line of implementation until the plan is approved.
